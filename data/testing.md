# Top Books on Testing

- **[Test-Driven Development: By Example](https://www.amazon.com/Test-Driven-Development-Kent-Beck/dp/0321146530)** - *Kent Beck* - *2002* - TDD from the inventor of TDD — walks through the process step by step with real code.
- **[Growing Object-Oriented Software, Guided by Tests](http://www.growing-object-oriented-software.com/)** - *Freeman & Pryce* - *2009* - The best book on TDD in practice, showing how tests shape real architecture.
- **[The Art of Unit Testing, 3rd Ed.](https://www.manning.com/books/the-art-of-unit-testing-third-edition)** - *Roy Osherove* - *2019* - Practical patterns for writing tests that are maintainable and trustworthy.
- **[Software Engineering at Google](https://abseil.io/resources/swe-book)** - *Winters, Manshreck, Wright* - *2020* - How Google thinks about testing, code review, maintainability, and scale — free online.
- **[xUnit Test Patterns](https://www.manning.com/books/xunit-test-patterns)** - *Gerard Meszaros* - *2007* - Refactoring test code and test patterns; the testing classic.
